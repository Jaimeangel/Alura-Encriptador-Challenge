# Proyecto: Venta Interna de Equipos Tecnológicos

## Objetivo del proyecto

Brindar un beneficio a los empleados mediante la venta de equipos
tecnológicos depreciados u obsoletos que normalmente serían enviados a
disposal. El objetivo no es generar un beneficio económico para la
empresa, sino facilitar el acceso a estos equipos a precios favorables,
manteniendo un proceso sencillo y con una carga administrativa mínima
para el equipo de IT.

------------------------------------------------------------------------

# Fase 1: Preparación e Inventario

Objetivo: Preparar el inventario que será ofrecido a los empleados y
validar la información mínima necesaria para ejecutar la venta.

Actividades principales:

-   Identificar los equipos que participarán en la primera ronda de
    venta.
-   Priorizar dispositivos de menor complejidad operativa:
    -   Dockings.
    -   Cámaras web.
    -   Monitores.
    -   Desktops (según disponibilidad).
-   Revisar el inventario existente en Excel.
-   Validar físicamente los activos cuando sea necesario.
-   Confirmar el activo fijo o etiqueta de identificación.
-   Verificar accesorios incluidos con cada dispositivo.
-   Registrar observaciones relevantes sobre el estado del equipo.
-   Preparar los equipos desde el punto de vista técnico cuando aplique.

Nota: Este ejercicio también representa una oportunidad para depurar y
fortalecer el inventario actual de IT.

------------------------------------------------------------------------

# Fase 2: Venta Interna Simplificada

Objetivo: Ejecutar una campaña interna sencilla, transparente y fácil de
administrar.

Proceso propuesto:

1.  Publicación general
    -   Enviar una comunicación oficial a toda la empresa.
    -   Utilizar apoyo del equipo de Comunicaciones o Marketing.
    -   Incluir imágenes de los productos, descripción general y
        precios.
    -   Compartir un enlace a un formulario de inscripción.
2.  Registro de interés
    -   Los empleados diligencian el formulario indicando los productos
        que desean adquirir.
    -   Se asume que el empleado tiene una intención real de compra.
3.  Selección por orden de llegada
    -   Cada producto tendrá un cupo definido.
    -   Ejemplo: 50 Dockings disponibles.
    -   Los primeros registros válidos avanzarán al siguiente paso.
    -   Los registros posteriores quedarán como lista de espera.
4.  Comunicación a seleccionados
    -   Los empleados seleccionados recibirán instrucciones para
        continuar con el proceso.
    -   Se indicarán fechas, horarios y ubicación para realizar pruebas
        de compatibilidad.
5.  Jornada de validación
    -   Los empleados podrán acercarse a la oficina para probar el
        producto.
    -   Podrán llevar su laptop personal para verificar compatibilidad
        cuando aplique.
6.  Pago
    -   Los empleados que deseen continuar realizarán el pago siguiendo
        las instrucciones definidas junto con Finance.
    -   Posteriormente enviarán el soporte correspondiente.
7.  Entrega
    -   IT realizará la entrega del equipo.
    -   Se registrará internamente el activo entregado y el empleado
        receptor.
8.  Gestión de remanentes
    -   Si algunos seleccionados no completan la compra, se podrá
        contactar a las personas que quedaron en lista de espera.
    -   Alternativamente, podrá realizarse una nueva ronda de
        comunicación interna.

------------------------------------------------------------------------

## Principios del proceso

-   Mantener la simplicidad operativa.
-   Evitar reglas complejas o excepciones innecesarias.
-   Minimizar la carga administrativa para IT.
-   Garantizar transparencia y equidad mediante el orden de inscripción.
-   Ajustar el proceso en futuras rondas según la experiencia obtenida.
